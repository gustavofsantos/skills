# Thinking Lenses — Reference

Five lenses. Apply one at a time. Stop when the lens has done its job.

---

## Lens 1 — The Iceberg

**When:** Problem keeps recurring. Proposed solution feels like a patch. Stuck on symptoms.

**Protocol:**

```
1. EVENT — What is the observable problem right now?
2. PATTERN — Is this a one-time event or has it happened before? What trends lead here?
3. STRUCTURE — What relationships, decisions, or constraints produce this pattern?
4. MENTAL MODEL — What beliefs underpin those structures?
```

Deeper = more leverage. A fix at the event level patches. A fix at the mental model level changes the system.

**Prompt:** "We seem to be working at the event level. Want to go deeper with the Iceberg?"

---

## Lens 2 — Systemic Reasoning Check

**When:** Before flushing. When reasoning feels thin or is being resisted.

**Protocol — four dimensions:**

```
RELIABLE — Are the reasons true? What evidence? What would falsify it?
RELEVANT — Are the reasons connected to this specific context, not someone else's?
COHESIVE — Do the reasons build on each other, or are they a loose pile?
COGENT — If all reasons are true, does the conclusion follow?
```

**Prompt:** "Before we flush this, want to run a quick reasoning check? It tends to catch what confidence hides."

---

## Lens 3 — What If We Do Nothing?

**When:** Change is assumed necessary without examining the cost of not changing.

**Protocol:**

```
1. Follow the status quo to its end — what happens concretely?
2. Does that outcome actually matter? By when? To whom?
3. Compare cost of action vs. inaction — is the change justified?
4. What does this reveal about the real problem?
```

**Prompt:** "We've been treating change as given. Want to spend a few minutes on 'what if we do nothing?'"

---

## Lens 4 — Seven Pattern Questions

**When:** Sociotechnical problem — people and systems both involved. Conversation is only on the technical side.

**Protocol — work through each, skip those that clearly don't apply:**

```
1. HOW DOES INFORMATION FLOW? Where does it originate, get stuck, get lost?
2. WHAT ARE THE EVENTS? What triggers changes? What causes what?
3. WHAT ARE THE BOUNDARIES? Are they where they should be, or historical accidents?
4. WHAT ARE THE BUILDING BLOCKS? Right granularity for the problem?
5. WHAT IS THE DELIVERY PROCESS? Where does work slow down or break?
6. HOW ARE PEOPLE ORGANIZED? Does team structure match system structure?
7. HOW IS DISCOURSE STRUCTURED? Who has voice? How are decisions made?
```

Gaps between answers are signal.

**Prompt:** "The technical side seems clear. Want to run the seven pattern questions to check whether the organizational side is aligned?"

---

## Lens 5 — What Is Braided Here?

**When:** Evaluating a design choice or tool. A change in one place keeps forcing changes elsewhere.

**Protocol:**

```
1. NAME THE CONCERNS — List distinct responsibilities or decisions. Don't group yet.
2. TRACE THE BRAID — Which concerns does this choice fuse? If I change A, does B move too?
3. COST OF SEPARATION — What would it take to keep these independent? What's given up?
4. ACCIDENT OR TRADE-OFF? — Was this coupling chosen deliberately, or did it arrive with a default?
```

A braid you chose is a trade-off. A braid you inherited without noticing is a liability.

**Prompt:** "We're evaluating a design choice. Want to run 'What Is Braided Here?' and look at what it actually couples?"

---

## Natural sequences

- **Iceberg → Braided:** Iceberg finds *where* the leverage is; Braided finds *what* is tangled there.
- **Reasoning Check → Do Nothing:** Reasoning Check exposes a gap; Do Nothing tests whether it actually matters.
- **Braided → Seven Patterns:** Braided reveals a coupling; Seven Patterns checks if the team structure mirrors it.
